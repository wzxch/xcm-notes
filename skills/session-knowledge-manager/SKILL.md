# 会话知识管理工具

将对话内容整理成结构化笔记并持久化保存到 GitHub。

## 功能

### 1. 生成摘要

从对话历史中提炼关键信息，生成结构化 Markdown。

**触发方式：**
- "生成摘要"
- "总结一下"
- "整理这段对话"

**输出：**
- 临时文件路径
- 统计信息（核心概念数、要点数、代码示例数、易错点数）

### 2. 保存笔记

将指定文件持久化到 GitHub 仓库。

**触发方式：**
- "保存到 GitHub"
- "保存笔记"
- "存到仓库"

**参数规则：**
- **文件路径（必填）：** 用户指定，未指定时反问补充
- **主题名称：** 用户指定优先，未指定时从文件内容推断

**流程：**
1. 读取指定文件
2. 确定主题名称
3. 内容适配为标准格式
4. 检测历史文件，存在则结构化合并
5. 自检内容
6. 创建分支 → commit → push → 创建 PR

**输出：**
- PR 链接
- 操作类型（新增/更新）
- 自检结果

### 3. 检索笔记

从 GitHub 拉取最新笔记，根据关键词检索内容。

**触发方式：**
- "搜索笔记"
- "找一下..."
- "查看笔记"（带关键词时检索，不带时列出）

**参数：**
- 关键词（支持主题名和内容搜索）

**输出：**
- 匹配结果列表
- 内容片段预览

### 4. 笔记检查

对笔记内容进行严格自检。

**触发方式：**
- "检查笔记"
- "审核内容"
- "严格自检"

**检查维度：**
- 技术准确性：概念、原理、数据是否有错误
- 表述严谨性：描述是否过于绝对、是否有歧义
- 完整性：关键细节是否遗漏、边界条件是否说明
- 时效性：内容是否过时
- 常见误区：是否忽略初学者易犯错误

**输出：**
- 整体评价
- 问题列表（严重/警告/建议）
- 修正建议

## 笔记格式

采用"总-分-总"结构：

```markdown
# 主题名称

## 概述

【总】用 1-2 句话概括文档核心内容。例如：本文档讨论了 Redis 集群方案，包括主从复制、哨兵模式和 Cluster 模式三种实现方式。

## 核心概念
- **概念1**：说明

## 要点总结

### 1. 要点一
详细说明...

### 2. 要点二
详细说明...

## 代码示例

### 示例 1
```代码片段
```

## 易错点
1. 易错点1

## 总结

【总】对各章节核心内容进行提炼，或强调全文最关键的信息。例如：Redis 集群方案的选择取决于业务规模——主从复制适合读多写少的小型应用，哨兵模式提供高可用性，Cluster 模式则支持大规模数据分片。

---
*最后更新：YYYY-MM-DD*
```

## Merge 策略

| 章节 | 策略 |
|:---|:---|
| 概述 | 智能合并，保留最新版本 |
| 核心概念 | 去重追加 |
| 要点总结 | 去重追加，重新编号 |
| 代码示例 | 追加，重新编号 |
| 易错点 | 去重追加 |
| 总结 | 智能合并，保留最新版本 |

## 配置

需要配置 git-repo-manager 的 `config.json`：

```json
{
  "repoUrl": "https://github.com/username/repo",
  "localPath": "/path/to/local/repo",
  "token": "ghp_xxxxxxxx"
}
```
